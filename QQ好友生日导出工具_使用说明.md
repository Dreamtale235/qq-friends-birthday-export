# QQ 好友生日导出工具 — 使用说明 & 可能存在的问题

## 环境准备

```bash
pip install -r requirements.txt
playwright install chromium
```

Python 版本要求 3.10+，安装时勾选 tcl/tk（Tkinter GUI 支持）。

## 使用方式

| 模式 | 命令 | 说明 |
|------|------|------|
| GUI | `python main.py` | 图形界面，有进度条和日志 |
| 命令行 | `python main.py --cli` | 终端模式，直接输出结果 |

首次运行会打开浏览器并要求**用手机 QQ 扫描二维码登录**，登录成功后会话会自动保存到 `sessions/` 目录，下次无需重复扫码。

## 可能存在的问题

### 1. 登录相关

- **扫码超时**：默认等待 3 分钟，超时后需重试
- **账号风控**：频繁登录可能触发 QQ 安全验证，需在手机上确认
- **会话过期**：QQ 邮箱会话有效期有限（通常几天），过期后需重新扫码
- **无 QQ 账号**：必须有一个能登录 QQ 邮箱的 QQ 号

### 2. 数据源相关

- **未订阅好友生日日历**：QQ 邮箱需提前在「日历 → 日历分类」中勾选「好友生日」，否则日历页不会显示任何生日数据
- **好友未公开生日**：只有填写了生日信息且设置为可见的好友才会出现在日历中
- **好友数量上限**：QQ 邮箱生日日历仅显示一定数量的好友，超出部分可能不展示

### 3. 页面结构变化

- **QQ 邮箱改版**：本工具依赖 `.grid-schedule-bar-subject`、`.grid-cell.currentMonth` 等 CSS 选择器定位生日数据。若 QQ 邮箱前端改版，可能导致解析失败（提取 0 条）
- **修复方式**：运行后在 `data/` 目录查看 `month_*.html` 文件，找到实际的生日 DOM 结构，修改 `crawler.py` 中的选择器

### 4. 运行环境

- **Playwright 浏览器**：首次需运行 `playwright install chromium` 下载 Chromium 内核，约 150MB，需要稳定的网络
- **Windows 编码**：好友昵称含特殊字符（如 ´͈ ᵕ `͈ ◞♡）时，命令行预览可能显示乱码，但 CSV 文件不受影响（UTF-8 with BOM）
- **杀毒软件**：部分杀毒软件可能拦截 Playwright 的自动化浏览器操作

### 5. 数据准确性

- **同一天多个生日**：同一日期有多个好友生日时，因 schedule bar 堆叠可能漏掉个别条目
- **跨月边界**：月末/月初的生日（如 1 月 31 日、2 月 1 日）可能被归到相邻月份
- **农历/公历**：QQ 邮箱生日日历可能混合农历和公历，本工具统一按公历处理
- **年份未知**：导出的 CSV 中 `birth_year` 字段为空，QQ 邮箱日历不提供出生年份

### 6. 导出文件

- **CSV 路径**：导出文件位于 `data/qq_friends_birthdays_YYYY-MM-DD.csv`
- **Excel 打开乱码**：用 Excel 直接打开 CSV 可能中文乱码，改用 WPS 或导入方式打开（数据 → 从文本/CSV → UTF-8）

### 7. 调试方法

- 运行时会自动在 `data/` 目录保存每月 HTML 和截图
- 日志文件位于 `logs/`，`error_*.log` 专门记录错误
- 若爬取结果为 0，查看 `data/month_05.html` 确认日历 DOM 是否存在
